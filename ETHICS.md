# Ethics & Responsible Use Policy

## 1. Purpose & Intent
The **Agentic AI Red Teaming Project** was developed by Rohan Chandel for educational, research, and authorized security testing purposes only. The primary goal of this project is to explore the intersection of Large Language Models (LLMs) and automated network reconnaissance to improve defensive postures and security automation.

## 2. Authorized Testing Only
This tool must **NEVER** be used against any infrastructure, network, or device without explicit, written, and informed consent from the owner. 
* Users are encouraged to run this tool only in isolated lab environments (e.g., Virtual Machines, HTB, TryHackMe, or private local networks).
* Unauthorized use of this tool against third-party systems may be illegal and is a violation of the Australian Cybercrimes Act 1901 and similar international laws.

## 3. Ethical Guardrails
As the author, I advocate for the following "Rules of Engagement":
* **Do No Harm:** Automated scans should be configured to avoid causing Denial of Service (DoS) or disrupting critical services.
* **Data Privacy:** Any data captured during authorized testing must be handled securely and destroyed once the audit is complete.
* **Responsible Disclosure:** If a vulnerability is discovered using this tool, it should be reported to the affected party via their official Bug Bounty program or security contact before being made public.

## 4. Professionalism in Australia
This project aligns with the professional standards expected within the Australian cybersecurity industry (ACS/AISA). It is designed to showcase how AI can assist human operators in identifying risks, not to facilitate malicious activity.
